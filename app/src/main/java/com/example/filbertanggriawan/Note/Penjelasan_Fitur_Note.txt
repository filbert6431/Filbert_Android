Berikut adalah rangkuman dari fitur Catatan (Note) yang telah Anda buat hari ini:

1. NoteEntity (Data Model)
   - Berfungsi sebagai struktur tabel di database Room.
   - Menyimpan informasi berupa: id (otomatis), judul (title), isi (content), dan waktu pembuatan (createdAt).

2. NoteDao (Data Access Object)
   - Interface yang mendefinisikan cara aplikasi berinteraksi dengan database.
   - Memiliki fungsi untuk mengambil semua data (getAll), menambah data (insert), dan menghapus data (delete).

3. AppDatabase (Database Room)
   - Merupakan pusat koneksi database menggunakan library Room.
   - Menggunakan pola Singleton (getInstance) agar hanya ada satu koneksi database yang terbuka di seluruh aplikasi.

4. NoteAdapter (UI List Management)
   - Bertugas untuk menghubungkan data dari database ke tampilan daftar (RecyclerView).
   - Menangani tampilan tiap item catatan dan aksi klik, termasuk memunculkan dialog konfirmasi saat ingin menghapus catatan.

5. NoteFormActivity (Input Data)
   - Halaman khusus untuk menambah catatan baru.
   - Mengambil input dari pengguna (judul dan isi) lalu menyimpannya ke database menggunakan Coroutines (lifecycleScope.launch) agar tidak memberatkan performa aplikasi.

6. NoteFragment (Main Feature UI)
   - Halaman utama yang menampilkan daftar catatan.
   - Secara otomatis mengambil data terbaru dari database setiap kali halaman dibuka (onResume).
   - Menyediakan tombol tambah (FAB) untuk membuka NoteFormActivity dan menangani logika penghapusan data.

Secara keseluruhan, Anda telah berhasil mengimplementasikan sistem penyimpanan lokal (Local Persistence) menggunakan Room Database dengan pola arsitektur yang rapi.

### 2. ALUR KERJA (WORKFLOW)
1. **Membuka Daftar**: Saat `NoteFragment` dibuka, fungsi `fetchNotes()` dipanggil di dalam `onResume()`. Data diambil dari database dan dikirim ke `NoteAdapter` untuk ditampilkan.
2. **Menambah Catatan**:
   - Klik tombol FAB -> Buka `NoteFormActivity`.
   - Input Judul & Isi -> Klik Simpan.
   - Data disimpan ke database secara asinkron.
   - Activity ditutup (`finish()`), kembali ke Fragment, dan daftar diperbarui otomatis.
3. **Menghapus Catatan**:
   - Klik ikon tong sampah di item -> Muncul Dialog Konfirmasi.
   - Klik "Ya" -> Fungsi `deleteNote()` di Fragment dipanggil -> Data dihapus dari database -> Daftar diperbarui.

### 3. BARIS KODE PALING PENTING (PROMINENT CODE)

1. **`suspend fun` pada NoteDao**
   ```kotlin
   suspend fun insert(note: NoteEntity)
   ```
   *Mengapa Penting?* Keyword `suspend` menandakan fungsi ini berjalan di background thread. Operasi database berat tidak boleh dijalankan di UI thread karena bisa membuat aplikasi "freeze" (macet).

2. **`AppDatabase.getInstance(context)`**
   ```kotlin
   return INSTANCE ?: synchronized(this) { ... }
   ```
   *Mengapa Penting?* Ini memastikan hanya ada satu instance database. Membuka banyak instance database sangat berat bagi performa HP dan berisiko merusak data (data corruption).

3. **`lifecycleScope.launch { ... }`**
   ```kotlin
   lifecycleScope.launch {
       db.noteDao().insert(note)
   }
   ```
   *Mengapa Penting?* Ini adalah **Coroutines**. Ia mengelola siklus hidup asinkron. Jika Activity/Fragment ditutup saat proses simpan sedang berjalan, Coroutine ini akan dibersihkan secara otomatis, mencegah *memory leak*.

4. **`onResume()` untuk Update Data**
   ```kotlin
   override fun onResume() {
       super.onResume()
       fetchNotes()
   }
   ```
   *Mengapa Penting?* Memastikan data selalu paling baru. Setiap kali pengguna kembali dari halaman tambah catatan, Fragment akan otomatis "mengambil ulang" data terbaru dari database.

5. **`MaterialAlertDialogBuilder`**
   *Mengapa Penting?* Memberikan pengalaman pengguna (UX) yang profesional dan mencegah penghapusan data secara tidak sengaja.
